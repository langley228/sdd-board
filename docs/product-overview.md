# 產品概覽

## 專案定位

sdd-board 是一個把 OpenSpec 從 CLI 擴展為 Web 工作台的多專案平台。

它的角色不是單純包裝 CLI，而是把以下能力集中到同一個平台：

- 專案設定
- 多人互動發想
- AI 協作
- 定案確認
- 外部系統開票
- 後續工程流程串接

## 核心主軸

```text
先有專案設定
   ↓
人與 AI 共同發想
   ↓
定案
   ↓
直接開票
   ↓
ticket 銜接後續流程
```

## 產品目標

1. 提供比 CLI 更直覺的 OpenSpec Web 使用體驗
2. 讓多人可在同一個空間內討論與整理規格草稿
3. 讓 AI 直接參與發想、補全、摘要與草稿生成
4. 讓定案內容能直接進入正式工作流程
5. 以專案為單位管理 ticket、repository、AI 與 automation 設定

## 重要原則

- 專案是所有流程的起點
- 發想與定案先在平台內完成
- 定案後才開票產生 ticket
- ticket、repository、AI provider 都不能寫死單一廠商
- 技術選型先偏輕量，但保留後續擴充空間
