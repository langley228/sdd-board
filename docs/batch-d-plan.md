# Batch D 規劃

本文件用來收斂 `sdd-board` 第四批實作主線，也就是整體平台的穩定化、治理與補強工作。

## 對應 Change

- `sc-012-platform-hardening`

## 目標

Batch D 的目標不是擴大產品能力，而是讓前面 Batch A、B、C 已建立的能力變得更穩、更安全、更可維護。

## Batch D 聚焦內容

### 1. 安全補強

- project 邊界保護
- 機敏設定保護
- AI 外送邊界補強
- provider 憑證保護

### 2. 權限補強

- project 可見性與操作權限
- settings 頁面權限
- 定案、開票、後續流程操作權限

### 3. 測試補強

- 補強主流程測試
- 補強 provider adapter 邊界測試
- 補強失敗路徑與錯誤處理測試

### 4. 容器化與執行環境補強

- 確保專案可容器化
- 整理環境變數與部署假設
- 確認未來可延伸的執行方式

### 5. 可維護性補強

- 補齊文件同步
- 補齊 logging / error handling 規則
- 收斂模組邊界與責任分工

## Batch D 不做什麼

- 不重新定義主流程
- 不新增大量新功能
- 不把穩定化工作混成新功能擴張

## 成功條件

- 主流程已有足夠測試保護
- 敏感邊界更清楚
- 權限模型至少能支撐第一階段需求
- 專案可作為後續持續擴充的穩定基座

## 與前面批次的關係

```text
Batch A: 跑通主線
Batch B: 補強體驗
Batch C: 擴充平台能力
Batch D: 穩定化與治理
```

## 一句話總結

> Batch D 的目標，是讓 `sdd-board` 從「有方向、可運作」進一步走向「可維護、可擴充、可長期演進」。
